# 经验工厂 - 设计文档

## 概述

一个 AI 驱动的个人工作经验积累系统。解决"工作时间长了记性差、更新简历需要大量回忆整理"的痛点。核心思路不是被动记录，而是 AI 主动引导用户完成经验积累，将零散口述自动加工为结构化的经验资产。

**设计原则：**
- 使用尽可能简便，降低自觉性依赖
- AI 主动引导，用户只需回应
- 管道化加工，每条经验从捕获到可用形成闭环
- 先个人 MVP，架构预留多用户扩展空间

---

## 核心交互形态

系统提供三种交互形态，弹窗和侧边栏为"轻入口"，主应用负责深度加工：

### 1. 定时弹窗提醒
- 可配置时间点（如 17:30 / 21:00）
- 轻量弹窗小对话窗口
- AI 主动提问："今天主要做了什么？一句话就行"
- 用户一句话回复即可，不打断工作流

### 2. 侧边栏快记
- 全局可呼出（浏览器内先做组件，后续 Electron 支持全局快捷键）
- 自由文本输入，想到什么记什么
- 支持语音输入入口（v2）
- 不切换界面，即记即走

### 3. 主工作区
- 左侧导航 + 右侧内容区
- 视图：教练对话 / 经验卡片 / 原始记录 / 设置
- 承担所有深度浏览和加工功能

---

## 系统架构

四层管道 + 一个横跨各层的引导系统：

```
[捕获层] → [加工层] → [存储层] → [生成层]
   ↕            ↕            ↕
引导系统（持续运行的定时/提醒/质量检测）
```

### 捕获层 Capture
- 教练对话：结构化 AI 对话，引导式提问
- 碎片快记：一句话/一段话自由记录
- 专项复盘（v2）：项目结束后的模板化总结

### 加工层 Process（AI 管道）
1. 要点提取：从原始输入提取成果、技术、方案
2. 量化增强：模糊描述转化为具体数据指标表述
3. 技能归类：自动打标签，关联技能节点
4. 质量评分：评估经验卡片的完整度

### 存储层 Store
- 经验卡片：核心实体，包含完整的结构化经验数据
- 技能图谱：技能节点 → 经验卡片的关联网络
- 原始记录：可追溯、可编辑的原始输入文本

### 生成层 Generate
- 简历生成（v2）：输入 JD → 匹配经验卡片 → 生成
- 面试素材（v2）：STAR 法则答题素材
- 能力报告（v2）：技能雷达图 + 成长轨迹

### 引导系统 Guidance
- 每日轻问：可配置时间的弹窗提醒
- 每周复盘（v2）：深度结构化回顾
- 断档检测：检测到长时间未记录时主动提醒
- 质量提示：卡片信息不完整时建议补充

## 数据模型

### ExperienceCard（经验卡片）
| 字段 | 类型 | 说明 |
|---|---|---|
| id | UUID | 唯一标识 |
| title | string | 卡片标题，AI 自动生成 |
| background | string | 时间/项目/角色背景 |
| tech_solution | string | 技术方案描述 |
| quantifiable_result | string | AI 量化增强后的成果表述 |
| reflection | string | 引导补充的反思总结 |
| skill_tags | string[] | AI 自动打的技能标签 |
| quality_score | int | 完整度评分（0-100） |
| created_at | datetime | 创建时间 |
| processed_at | datetime | 最近一次 AI 加工时间 |
| source_record_id | UUID | 关联的原始记录 |
| source_updated_at | datetime | 原始记录最后编辑时间 |
| metadata | JSON | 预留扩展字段 |

### RawRecord（原始记录）
| 字段 | 类型 | 说明 |
|---|---|---|
| id | UUID | 唯一标识 |
| content | text | 原始输入内容 |
| source_type | enum | 对话 / 快记 / 复盘 |
| recorded_at | datetime | 记录时间 |
| updated_at | datetime | 最后编辑时间 |
| process_status | enum | 待加工 / 已加工 |

### SkillNode（技能节点）
| 字段 | 类型 | 说明 |
|---|---|---|
| id | UUID | 唯一标识 |
| name | string | 技能名称 |
| category | string | 技能类别 |
| card_count | int | 关联的经验卡片数 |
| proficiency | float | AI 计算的熟练度 |

---

## 核心数据流

1. **记录**：用户通过弹窗对话 / 侧边栏快记输入 → 存储为 RawRecord
2. **加工**：RawRecord 进入 AI 管道 → 提取/量化/归类 → 生成/更新 ExperienceCard
3. **查看**：在主应用浏览卡片和原始记录，支持按技能筛选、搜索
4. **编辑**：编辑 RawRecord → 标记 source_updated_at → 触发卡片重新加工
5. **提醒**：引导系统按配置时间触发弹窗提醒

## 技术栈

| 层 | 选型 | 说明 |
|---|---|---|
| 后端框架 | Python + FastAPI | 异步支持好，IO 密集的 AI 调用场景匹配 |
| LLM | 可配置（默认 Claude API） | 通过 Provider Adapter 抽象，切换模型不影响功能 |
| 数据库 | SQLite（MVP）→ PostgreSQL | SQLAlchemy ORM 做抽象，迁移透明 |
| 定时任务 | APScheduler | 进程内轻量调度 |
| 前端框架 | Vue 3 + Vite | 轻量、上手快 |
| 前端样式 | Tailwind CSS | 快速构建界面 |
| 通知 | Web Notification API | PWA 路径，后续 Electron 获得系统级能力 |

### LLM 可配置架构
- 定义 `LLMProvider` 抽象接口（`chat(prompt) -> response`）
- 每种模型独立实现（ClaudeProvider、GPTProvider、DeepSeekProvider 等）
- 管道代码仅依赖接口，不依赖具体模型
- 设置页面配置：模型名称 / API Key / Base URL，即时切换生效

## MVP 范围

### v1.0 包含
- 教练对话（主应用内）
- 碎片快记（侧边栏组件）
- AI 加工管道（要点提取 + 技能标签 + 量化增强）
- 经验卡片浏览（列表 + 搜索 + 技能筛选）
- 原始记录查看和编辑（编辑后触发卡片重新加工）
- 每日提醒弹窗（可配置时间，PWA 通知）
- LLM 可配置（设置页面）

### v1.0 不包含（后续迭代）
- 专项复盘模板（v2）
- 语音输入（v2）
- 简历生成（v2）
- 能力图谱可视化（v2）
- 面试素材生成（v2）
- 多用户/分享（v2）
- Electron 桌面打包（后续）
